Practices associated with using classes vs IDs

In web development, it is common to use classes and ID selectors to identify what attributes to style on a webpage. This blog post will show how we can distinguish the differences and how to apply them.

Classes

Classes are commonly used to style attributes that fall under the same category. When using classes, every attribute with the same name will be styled the same way. You implement a class in HTML by writing an attribute, followed by the class attribute, then the name of the class. The class attribute in css follows simplified writing, with a dot followed by the class name and {} brackets, and the content goes inside the brackets. IDs

IDs work similarly to classes, but instead of having multiple attributes categorized in one class, IDs are used in a single instance on a page, and cannot be reused for other attributes. When identifying and implementing a ID selector in HTML, the process follows the same steps as a class selector, but instead of using a fullstop in css, it is written with a # followed by the ID name and {}.
